Pillar 01 · Document Intelligence

Claude Vision reads every PDF.

Drag a W2, paystub, or bank statement onto the file. Claude Vision extracts every number, calculates AGI and monthly gross, flags large unidentified deposits, and writes the result back to the loan record. The processor stops typing. You stop waiting on the processor.

Every change. Every actor. Every timestamp.

Field level diffs auto write to an audit log on every insert, update, and delete. TRID, ECOA, and HMDA recordkeeping requirements satisfied without app code remembering. Status transitions captured separately for time in stage analytics. So when a borrower asks "where's my file?" you have a real answer in two seconds.

  • 25 month retention default · configurable
Spotlight · The PDF Drop Zone

Drop a PDF. Watch the file build itself.

This is the feature that closed every demo so far. The processor (or you, on an iPad with a borrower at the kitchen table) drags a stack of borrower docs onto the loan file. Claude Vision reads each one, classifies the doc type, extracts the structured data, and writes it back to the loan record. The matching condition auto clears. Zero clicks. Zero stare and compare. The borrower watches their file build itself in real time.

The audit log captures the AI confidence score and the model version, so a compliance officer can replay the extraction six months later and verify the math. Low confidence extractions get routed to a manual review queue instead of silently failing. You stay safe. You also stay fast.

  • ✓Income calc. AGI from W2 box 1, plus monthly gross from paystub gross divided by pay period count.
  • ✓Asset verification. Bank statement scan for any deposit over 50% of monthly income, auto flagged.
  • ✓Data integrity. Side by side PDF vs entered values. Discrepancies highlight in amber before submit.
  • ✓Condition clearing. Doc type W2 flips the "Most recent W2" condition to submitted automatically.
  • ✓OCR robustness. Rotated, faxed, multi page financial docs handled by Claude Vision natively.

Who built this

A submarine, a spreadsheet, and a frustrated loan officer.

I spent three years in the Navy, most of it on the USS Montana during dry dock in Newport News, Virginia. What you learn in that environment is that everything runs on procedure, verification, and trusting your instruments when your instincts are screaming something else.

I got medically separated and enrolled at Texas Tech for a BBA in IT Systems. While I was there I built Submarine Catalyst, a biotech PDUFA scanner. Same submarine aesthetic, same obsession with making the machine do the boring work. It taught me how to ship.

OriginFlow started because a loan officer friend in Texas wouldn't stop complaining about Encompass. The 60 second file loads. The shared Excel pipeline tracker. The $100k Encompass admin his branch couldn't afford. Every problem he described was a thing I could fix with a live database and an AI model that can read a paystub.

So I started building. The schema went up first. Then Claude Vision wired into the document pipeline. Then the dashboard. He's the first beta tester and the reason every feature exists. If it doesn't fix something a working LO has actually complained about, it doesn't make the cut.

By graduation in December 2026, I want OriginFlow LOS to be the leanest, fastest, cheapest alternative for independent loan officers across Texas. Then I expand. The aircraft carrier doesn't need to sink. I just need to be the better boat for the LOs it was never designed for.

A processor (or you, on an iPad with a borrower in front of you) drags a PDF onto the loan file. The doc gets stored in Supabase Storage and an extraction record is created. A serverless function picks it up, sends it to Claude Vision with a prompt scoped to the doc type (W2, paystub, bank statement, etc.), and parses the response into structured fields. The result lands back on the document record as JSONB, and a trigger fires to auto clear any matching condition. The AI's confidence score is recorded. Anything under a configurable threshold goes to manual review.
Row level security at the Postgres layer scopes every query to the user's branch. Even a leaked anon key cannot see another branch's data. Field level audit logging via database triggers covers every insert, update, and delete on compliance relevant tables (loans, borrowers, conditions, documents). SSN is never stored in plaintext, only a salted hash and last four. Full PII goes through Supabase Vault for envelope encryption when needed for credit pulls. The retention default matches TRID and ECOA recordkeeping windows (25 to 36 months). Configurable.
